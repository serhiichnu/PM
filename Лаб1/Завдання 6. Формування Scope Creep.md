### ЗАВДАННЯ 6. Формування Scope Creep

#### Scope Creep для MVP
На етапі MVP головна мета — довести технічну можливість ведення списків книг та обліку прочитаних сторінок. Будь-яка задача, що виходить за ці межі, є загрозою для дедлайну.

**Приклади Scope Creep:**
1. **Інтеграція з соцмережами (Автопостинг):** Можливість автоматично постити свої результати чи цитати у сторіз Instagram або Facebook.
   * *Чому це Creep:* Це вимагає розробки системи авторизації через сторонні API та генерації картинок, що ніяк не допомагає основному функціоналу (трекінгу) і суперечить концепції "анти-соцмережі".
2. **Складна візуалізація полиць (3D-анімація):** Створення анімованих книжкових полиць, де книги можна переставляти свайпами, з 3D-ефектом гортання сторінок.
   * *Чому це Creep:* Це технологічний «оверінжиніринг». Написання складного UI з'їсть весь час, який має бути витрачений на стабільну роботу бази даних книг.

**Наслідки для «Залізного трикутника»:**
* **Scope (Зміст) ↑**, що автоматично призводить до:
* **Time (Час) ↑** — команда не встигне здати базовий реліз MVP до передбаченої дати.
* **Quality (Якість) ↓** — через поспіх основний модуль збереження прогресу буде працювати з критичними помилками.

#### Scope Creep для MMP
На етапі MMP продукт виходить на ринок і має стабільно працювати для щоденного використання (стріки, нотатки, синхронізація).

**Приклади Scope Creep:**
1. **Вбудована "Читалка" (ePub/PDF reader):** Спроба дати можливість завантажувати і читати самі файли книг всередині додатка BookOff.
   * *Чому це Creep:* Це виходить за межі проєкту (Project Scope). Розробка власного рушія для рендерингу ePub-файлів, робота з шрифтами та закладками — це створення зовсім іншого продукту, що збільшить обсяг коду вдесятеро.
2. **Алгоритмічна стрічка новин (TikTok-стиль):** Розробка складної системи рекомендацій відгуків інших користувачів на основі Machine Learning.
   * *Чому це Creep:* Це вимагає створення складного бекенду для обробки великих даних (Big Data). Для MMP достатньо мати стабільні локальні нотатки та базовий пошук.

**Наслідки для «Залізного трикутника»:**
* **Cost (Вартість) ↑↑** — потреба у залученні додаткових розробників (Data Science або спеціалістів з рендерингу тексту) та розширенні серверних потужностей.
* **Time (Час) ↑** — запуск продукту на ринок відкладеться на місяці.

#### Scope Creep для MLP
На фінальному етапі ми впроваджуємо "Wow-ефект" (QA-режим, Дуелі, Shelf Scanner). Загрозою стає бажання впровадити технології заради технологій (хайп).

**Приклади Scope Creep:**
1. **Віртуальна реальність (VR Метавсесвіт бібліотеки):** Можливість одягнути окуляри Meta Quest і «походити» по віртуальній кімнаті зі своїми прочитаними книгами.
   * *Чому це Creep:* Це вимагає розробки на ігрових рушіях (Unity/Unreal Engine). Практична цінність для трекінгу — нульова, це марнотратство ресурсів.
2. **Eye-tracking (Відстеження погляду через камеру):** Спроба автоматично рахувати прочитані сторінки паперової книги, відстежуючи рух очей користувача через фронтальну камеру під час читання.
   * *Чому це Creep:* Це межує з науковою фантастикою. Нестабільна детекція погляду при поганому освітленні призведе до помилок у трекінгу, що повністю знищить довіру до додатка. Плюс, це швидко розряджає телефон.

**Наслідки для «Залізного трикутника»:**
* **Scope ↑↑** — межі проєкту розмиваються до рівня «науково-фантастичного фільму».
* **Quality (Якість) ↓↓** — складна та непотрібна логіка AI/VR плодить «важкі» баги. Замість продукту, який обожнюють (Loveable), виходить продукт, який гальмує і дратує користувача.
