Критичний ризик для MVP: Інтеграція з Google Books API (Smart-пошук)
Найбільш ризикованою характеристикою для старту є стабільна інтеграція зі стороннім API для пошуку та додавання книг. Оскільки це фундамент "Цифрової бібліотеки", будь-які проблеми з парсингом метаданих, лімітами запитів або відсутністю україномовних видань у базі унеможливлять швидке створення користувачем своїх списків. Без працюючого наповнення бібліотеки етап MVP втрачає свою головну цінність — можливість вести облік прочитаного.

Критичний ризик для MMP: Безконфліктна хмарна синхронізація (Reading Streak)
Найбільшим викликом для релізу MMP є забезпечення безперебійної синхронізації прогресу читання між пристроями, особливо після офлайн-сесій. Оскільки на цьому етапі впроваджується "Reading Streak" (трекінг безперервних днів читання), будь-який баг бази даних, який "обнулить" стрік користувача через помилку мережі, викличе сильне роздратування. Технічна складність полягає у правильному вирішенні конфліктів даних, щоб не допустити відтоку розчарованої аудиторії.

Критичний ризик для MLP: Сканер полиць (Shelf Scanner на базі Computer Vision)
Для емоційного релізу MLP найбільш ризикованою є розробка масового сканера корінців книг через камеру смартфона. Навчити ML-модель безпомилково розпізнавати корінці різної товщини, шрифтів та під різним кутом освітлення — це надзвичайно ресурсоємна задача, що вимагає залучення Data Science спеціалістів. Якщо точність розпізнавання буде низькою, ця «Wow-фіча» перетвориться на джерело роздратування і спалить левову частку бюджету та часу розробки.
