# 2.4 High‑level Acceptance Criteria для Epics

У цьому документі наведено по 3–5 High‑level Acceptance Criteria для кожного Epic з WBS.
Критерії сформульовані декларативно, без деталей імплементації.

---

## Epic 1.0 — UI + Навігація
| № | Acceptance Criterion |
| :--- | :--- |
| 1 | Користувач може перейти між основними розділами (бібліотека, пошук, прогрес, профіль) без втрати стану. |
| 2 | Система зберігає обраний розділ/контекст між сеансами (після перезапуску додатку). |
| 3 | Невпорядкований або неочікуваний стан навігації не призводить до падіння додатку (crash-free). |
| 4 | Невірна або відсутня авторизація перенаправляє користувача на екран логіну/реєстрації. |

---

## Epic 2.0 — Цифрова бібліотека (списки книг)
| № | Acceptance Criterion |
| :--- | :--- |
| 1 | Користувач може додавати книги до списків «Хочу прочитати», «Читаю», «Прочитано». |
| 2 | Зміни в списках зберігаються й відновлюються після повторного запуску додатку. |
| 3 | Користувач може переміщувати книгу між списками без втрати даних. |
| 4 | Доступ до бібліотеки обмежений лише авторизованим користувачам. |

---

## Epic 3.0 — Smart‑пошук (Google Books API)
| № | Acceptance Criterion |
| :--- | :--- |
| 1 | Користувач може виконати пошук книг за ключовим словом і отримати перелік результатів. |
| 2 | Система обробляє ситуації, коли API повертає помилку або відсутні результати, і показує відповідне повідомлення. |
| 3 | Користувач може додати знайдену книгу до своєї бібліотеки з результатів пошуку. |
| 4 | Виклики до API здійснюються тільки для авторизованих користувачів (або через захищений проксі). |

---

## Epic 4.0 — Автентифікація користувача
| № | Acceptance Criterion |
| :--- | :--- |
| 1 | Користувач може зареєструватися та увійти в систему, отримавши валідну сесію/токен. |
| 2 | Доступ до всіх функцій, що зберігають дані, вимагає валідної авторизації. |
| 3 | Невдалий логін повідомляє користувача про помилку (невірні дані) без витоку інформації. |
| 4 | Система автоматично завершує сесію після закінчення строку дії токена. |

---

## Epic 5.0 — Трекер прогресу читання
| № | Acceptance Criterion |
| :--- | :--- |
| 1 | Користувач може задати та оновити прогрес читання (сторінки/відсоток) для кожної книги. |
| 2 | Прогрес зберігається та відображається при наступному вході в додаток. |
| 3 | Прогрес не може бути збережений без авторизації. |
| 4 | Система коректно обробляє зміни прогресу (навіть якщо користувач змінює дані паралельно на іншому пристрої). |

---

## Epic 6.0 — Бекенд + Інфраструктура (DB, API, CI/CD)
| № | Acceptance Criterion |
| :--- | :--- |
| 1 | API відповідає на запити з визначеним SLA (наприклад, відповідає за < 2 сек у нормальних умовах). |
| 2 | Дані користувача зберігаються в базі даних і відновлюються після перезапуску сервера. |
| 3 | Система має автоматичну збірку та деплой (CI/CD), що дозволяє оновлювати середовище без ручного втручання. |
| 4 | Невдалі запити / помилки логуються для подальшого аналізу. |
