Завдання 6. Аналіз точності (MRE)

Sprint Backlog (обрано для аналізу):
- US №5 — "Додати книгу до списку \"Хочу прочитати\"" — Estimate E5 = 32.66 год (5 SP)
- US №9 — "Шукати книги за назвою" — Estimate E9 = 30.66 год (8 SP)

Сценарій Actual: O (оптимістичний) + 2 год.
- US №5: O_sum = 23 год → Actual_5 = 23 + 2 = 25 год
- US №9: O_sum = 22 год → Actual_9 = 22 + 2 = 24 год

MRE = |Actual − Estimate| / Estimate
- US №5: MRE = |25 − 32.66| / 32.66 = 7.66 / 32.66 = 0.235 ≈ 23.5% (Overestimation)
- US №9: MRE = |24 − 30.66| / 30.66 = 6.66 / 30.66 = 0.217 ≈ 21.7% (Overestimation)

Average MRE = (23.5% + 21.7%) / 2 = 22.6%

Найбільше відхилення: US №5 (23.5%). Причина: "Lucky Strike" — знайдена готова бібліотека значно скоротила роботу (технічний фактор); команда консервативно оцінила задачі, тож Actual < Estimate.

Коротка резолюція:
Оскільки наше average MRE ≈ 22.6%, ми маємо систематичну переоцінку (Overestimation). Обираємо Варіант А — корекцію 1 SP: перераховуємо коефіцієнт на основі фактичних годин.

NewRatio = ΣActual_hours / ΣStoryPoints = (25 + 24) / (5 + 8) = 49 / 13 ≈ 3.77 год/SP

Рекомендації коротко:
- Використовувати NewRatio ≈ 3.8 год/SP для найближчих оцінок або залишити стандартну конверсію і планувати невеликий буфер (≈10–20%) для випадкових змін.
- До наступного планування додати spike/перевірку reuse-бібліотек, щоб зменшити варіативність оценок.
